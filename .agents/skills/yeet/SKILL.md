# /yeet

Spellbook worktree → one or more conventional commits → remote. One command.
Executive authority. No approval gates.

## Stance

1. **Act, do not propose.** Stage what belongs, leave out what doesn't,
   delete debris, split logically, push. Escalate only on a Refuse Condition.
2. **Clean tree is the deliverable.** Not done while `git status --short`
   shows any path. Resolve every path by commit, ignore, move, or delete.
3. **Reviewability is the product.** Three focused commits beat one
   2,000-line "wip". Split on semantic boundaries even when the tree was
   built in one session.
4. **Never lose work.** Untracked scratch that might be in-flight thinking
   gets moved, not deleted, unless it's unambiguous debris.
5. **Conventional Commits, matched to this repo's log.** Spellbook has no
   `commitlint.config.*` — conventions are by example. `git log -20` is the
   source of truth.

## Modes

- Default: classify → stage → split → commit → push.
- `--dry-run`: print the plan (classes, commit boundaries, messages, push
  target). No git writes.
- `--single-commit`: skip the split pass; everything signal-class becomes
  one commit.
- `--no-push`: commit locally; leave the push for the user.

## Process

### 1. Read the worktree holistically

- `git status --short` (untracked + modified + staged; do not truncate)
- `git diff --stat` + `git diff --stat --cached`
- `git log -20 --oneline` (scope + type pattern on this branch)
- `git rev-parse --abbrev-ref HEAD` (push target)
- `git status` for rebase / merge / cherry-pick (see Refuse Conditions)

If the tree is clean, say so and exit.

### 2. Classify every path

| Class | Meaning | Action |
|---|---|---|
| **signal** | Intended work (skill edits, agent edits, CI, bootstrap, scripts, harness configs, backlog moves) | Include in a commit |
| **debris** | `.DS_Store`, editor swap (`*.swp`, `*.swo`, `*~`, `.#*`), `*.orig`, `node_modules/`, `__pycache__/`, `*.pyc` that slipped the gitignore | Delete |
| **drift** | Unrelated work from another concern — e.g. a stray edit in `skills/refactor/` while the session was about `skills/tailor/` | Own commit with its own scope, or move out of repo, or durable ignore |
| **evidence** | Artifacts under `.spellbook/deliver/<ulid>/`, `.groom/review-scores.ndjson`, ad-hoc logs | Pre-commit blocks force-adds of `state.json` / `receipt.json`; do not fight it. Other evidence: move out of repo or durable-ignore |
| **scratch** | Half-written notes, TODO files, planning docs not under `backlog.d/` | Move to a vault outside the repo, or delete if trivial |
| **secret-risk** | Plausible credential / token / `.env`-like content | REFUSE the commit, surface to user |
| **derived** | `index.yaml` (regenerated by pre-commit), `skills/.external/` (gitignored), copied `harnesses/claude/settings.json` | Do not stage manually; let the hook do it |

**Spellbook-specific heuristics:**

- `index.yaml` is **never** human-edited. If it appears in the working
  tree, either the pre-commit hook already regenerated it (leave it; the
  hook will auto-stage on the next commit) or someone touched it by hand
  (revert). Don't craft a standalone `chore: regen index.yaml` commit.
- `.spellbook/deliver/<ulid>/state.json` and `receipt.json` are agent-
  written and blocked by pre-commit. If they're in the diff, they were
  force-added — unstage them and confirm `.gitignore` covers
  `.spellbook/deliver/`.
- `skills/.external/` is the embeddings sync destination, gitignored.
  Never commit.
- `harnesses/claude/settings.json` is copied by bootstrap, not symlinked.
  Edits here land in commits normally; note that consumers need to re-run
  `./bootstrap.sh` to pick them up (post-commit runs bootstrap locally).
- Secret patterns to grep the diff for: `-----BEGIN.*PRIVATE KEY-----`,
  `api[_-]?key.*=.*["'][^"']{20,}`, `(AKIA|ghp_|github_pat_|sk-)[A-Za-z0-9]{16,}`.
  Match → refuse.

### 3. Group signals into commits — spellbook scope map

**Type in rotation:** `feat`, `fix`, `refactor`, `docs`, `chore`, `test`,
`backlog`, `doctrine`. No `wip`, `misc`, `update`.

**Scope is the primary affected skill, agent, or module.** Observed
vocabulary on this branch and its parents:

| Path touched | Commit shape |
|---|---|
| `skills/<name>/SKILL.md` or `skills/<name>/references/...` | `feat\|fix\|refactor(<name>): …` — e.g. `refactor(flywheel): thin-harness — scripts out, judgment in` |
| `agents/<name>.md` | `feat\|fix\|refactor(<name>): …` (scope = agent name) |
| `harnesses/shared/AGENTS.md` (doctrine / principles) | `doctrine(harness): …` or `doctrine(<topic>): …` — e.g. `doctrine(harness): cross-harness first` |
| `harnesses/{claude,codex,pi,...}/...` configs | `feat\|fix\|chore(harness): …` |
| `bootstrap.sh` | `feat\|fix\|chore(bootstrap): …` |
| `scripts/<tool>.sh` or `.py` behavior change | `fix\|feat(<concern>): …` where concern is the affected skill or subsystem (e.g. `fix(ci): exclude skills/.external from CI source context`) |
| `scripts/<tool>` pure housekeeping | `chore(scripts): …` |
| `.githooks/*` | `chore(hooks): …` or `fix(hooks): …` |
| `ci/src/spellbook_ci/main.py` (gate logic) | `feat\|fix(ci): …` |
| `backlog.d/NNN-*.md` new / shaped / notes | `backlog(NNN): <verb> …` — e.g. `backlog(029): add Implementation Notes` |
| `backlog.d/NNN-*.md` → `backlog.d/_done/NNN-*.md` | `backlog: close NNN …` or `backlog(NNN): ship` |
| `registry.yaml` | `feat\|chore(registry): …` |
| `dagger.json` | `chore(dagger): …` |

**Grouping rules:**

- **One concern per commit.** Separate a skill body edit from a hooks fix
  from a bootstrap change.
- **Tests ride with their code.** Frontmatter-check additions + the
  `scripts/check-frontmatter.py` change ship as one commit.
- **Refactors before features.** If the diff contains both a pure refactor
  and a feature built on it, commit the refactor first (sane bisect).
- **Doctrine commits are standalone.** `harnesses/shared/AGENTS.md`
  principle additions are their own commit with `doctrine(...)`; don't
  bundle with an unrelated skill edit. Precedent: `985da20`, `0a0494d`,
  `f696c73`.
- **Backlog close != skill ship.** When a backlog item closes because a
  skill shipped, commit the skill as `feat(<skill>): …` first, then the
  backlog move as `backlog: close NNN …` (or fold into the skill commit
  only if the item is genuinely trivial — precedent `b672b64`).
- **Cross-cutting scope is its own commit.** Gate changes in
  `ci/src/spellbook_ci/main.py` are `feat\|fix(ci): …`, never bundled into
  a skill commit that happens to touch a CI regex.
- **Do not split the pre-commit index regen.** When `skills/<name>/SKILL.md`
  changes, the pre-commit hook auto-regenerates and auto-stages
  `index.yaml`. It lands inside the same commit as the skill edit. Never
  produce a separate `chore: regen index.yaml` commit.
- **Carmack's stapled-PR rule.** "X and also Y" → two commits.

With `--single-commit`, skip grouping; all signal-class paths become one
commit. Choose the type/scope of the dominant concern.

### 4. Write commit messages

Imperative subject, scope in parens, body explains **why**, not what.

```
<type>(<scope>): <imperative subject under 72 chars>

<optional body: the constraint, the incident, the reason. Wrap at 72.>

<optional footer: refs, Co-Authored-By>
```

Subject rules (observed in the log):

- Imperative voice ("add", "close", "resolve"), not past tense or
  gerund.
- No trailing period.
- Em-dash to sub-title is acceptable: `refactor(flywheel): thin-harness
  — scripts out, judgment in`.
- No PR / issue numbers in the subject except via the `backlog(NNN)` scope.

Body rules:

- Omit entirely when the subject self-explains.
- Explain the constraint or rationale ("why this over the alternative").
  Do not restate the diff.
- Wrap at 72 columns.

Co-author: match what the existing log uses on this branch. Grep
`git log -30 | grep -i co-authored-by` before deciding; do not invent a
line the project hasn't used.

### 5. Stage, commit, push

- `git add` the signal paths for the current group only, path-by-path.
  Never `git add -A` at repo root.
- `git commit` per group. Let hooks run:
  - **pre-commit** regenerates and auto-stages `index.yaml` when
    `skills/` or `agents/` change, and runs
    `scripts/check-harness-agnostic-installs.sh`. If it blocks the
    commit, the install wording check failed — fix the wording, do not
    `--no-verify`.
  - **pre-commit** also blocks force-added `.spellbook/deliver/<ulid>/
    state.json` or `receipt.json`. Unstage them; don't bypass.
  - **post-commit** re-runs `./bootstrap.sh` when skill/agent content
    changed. Expect fresh symlink output; it's not an error.
- After the final commit: `git push`. If no upstream,
  `git push -u origin <branch>`.
- If the push is a **fast-forward to `master`** from a local topic branch
  (pattern: deliver-on-master flow), `pre-merge-commit` does not fire —
  FF merges leave no merge commit. The verdict gate is bypassed by
  construction. If the user wanted a reviewed merge, they wanted a
  non-FF merge, which is a different workflow; `/yeet` is not responsible
  for that decision.
- If a **non-FF merge** happens as part of the push (rare — /yeet doesn't
  create merge commits itself), `pre-merge-commit` runs the verdict gate;
  failure means `/code-review` never produced a verdict for this branch.
  Surface the failure, do not set `SPELLBOOK_NO_REVIEW=1`.
- On push rejection (upstream moved): `git pull --rebase` once, retry
  push. Second rejection → stop.
- After push, rerun `git status --short --untracked-files=all`. Visible
  entries mean not done.

### 6. Report

What got committed (one line per commit: sha, type(scope), subject).
What got deleted / moved / ignored and why.
Push target + result.
Final worktree status.

## Refuse Conditions

Stop and surface instead of committing:

- `.git/MERGE_HEAD`, `.git/CHERRY_PICK_HEAD`, `rebase-*` dir exists.
- Diff contains unresolved conflict markers.
- Any path classified `secret-risk`.
- HEAD is detached.
- Branch is `master` AND push would write to `master` AND the user did
  not explicitly request it. (Spellbook's base branch is `master`;
  topic branches are `feat/*`, `fix/*`, `chore/*`, `docs/*`, `refactor/*`.
  Force-pushing to `master` is never acceptable without explicit request.)
- >500 changed files AND no obvious semantic grouping — ask before
  assuming it's one session's work.
- `.spellbook/deliver/<ulid>/state.json` or `receipt.json` is staged.
  Something force-added them; unstage and confirm the gitignore.

## Safety rails (never)

- Never force-push.
- Never `--no-verify`.
- Never `git add -A` at the repo root without classifying first.
- Never `git clean -fdx`.
- Never commit files matching the secret patterns above.
- Never hand-edit `index.yaml` to make it "match" — rerun the hook.
- Never declare success while `git status --short` shows entries.

## Gotchas

- **Scope matches precedent, not taxonomy.** The log says `feat(tailor)`
  not `feat(skills/tailor)` or `feat(tailor-skill)`. Grep `git log -20`
  for the scope shape before inventing one.
- **Doctrine changes are load-bearing.** `harnesses/shared/AGENTS.md`
  edits use `doctrine(harness)` or `doctrine(<topic>)` — they are the
  principles every harness reads. Never mis-scope them as `docs(harness)`.
- **`backlog(NNN)` is the convention for ticket-scoped work.** Status
  changes to `backlog.d/NNN-*.md` belong under this scope. Bare
  `backlog:` (no number) is acceptable when closing/moving several at
  once — precedent `0aee4ab`, `b672b64`.
- **Pre-commit auto-stages `index.yaml`.** After your `git add` for a
  skill edit, running `git commit` will add `index.yaml` to the same
  commit. This is correct and expected. Do not pre-stage `index.yaml`
  yourself; do not split it into a separate commit.
- **Post-commit re-runs bootstrap.** The symlink output after every
  skill/agent commit is the `post-commit` hook, not a failure mode.
- **`check-harness-agnostic-installs.sh` rejects Claude-only wording**
  in `skills/seed/` and `skills/tailor/`. If the hook blocks your commit,
  the skill body said "install into `.claude/`" where it should say
  `.agents/` or enumerate all three harnesses. Fix the wording, re-commit.
- **Untracked dirs need directory-by-directory classification.**
  `git add <dir>` recurses; `git add` without the dir path silently
  skips untracked directories.
- **"Tidy" is not refactor.** `/yeet` stages and commits. It does not
  edit source to make the diff prettier; that's `/refactor`.
- **Co-author line match.** Grep `git log -30` for `Co-Authored-By`;
  spellbook commits are typically un-co-authored unless a session did
  real implementation work.
