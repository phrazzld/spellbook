# Spellbook — Repo Brief

**The shared spine every rewriter reads.** Downstream rewriters cite anchors
from this file verbatim (no parallel vocabulary).

## Vision & purpose

Spellbook is a **harness library**: one checkout, three harnesses (Claude
Code, Codex, Pi). It ships focused workflow skills, specialized agents, and
per-harness configs to `~/.claude`, `~/.codex`, `~/.pi` via `bootstrap.sh`.
The product is *the agent-shaped coating* — skills (judgment-encoding
markdown), agents (scoped personas), hooks, and settings. End-users invoke
those skills in their own projects.

This is a library-of-libraries: skills are consumed by `/tailor` and `/seed`
into per-repo harnesses. Spellbook itself has no app, no service, no UI —
its "runtime" is whichever harness reads the symlinked `SKILL.md` files.

## Stack & boundaries

- **Skills** (`skills/<name>/SKILL.md`) — judgment, not procedure. <500
  lines, frontmatter-triggered. 26 skills current.
- **Agents** (`agents/<name>.md`) — scoped personas with tool restrictions.
  10 agents (philosophy bench: ousterhout, carmack, grug, beck, critic,
  planner, builder; a11y triad).
- **Harnesses** (`harnesses/{claude,codex,pi,factory,gemini,shared}/`) — per-
  harness configs and hooks. `harnesses/shared/AGENTS.md` is THE principles
  file, symlinked to every harness.
- **CI** (`ci/src/spellbook_ci/main.py`) — Dagger module, Python 3.12, 12
  parallel gates, local-first. `dagger.json` names the module; `ci/sdk/` is
  the bundled Dagger Python SDK.
- **Bootstrap** (`bootstrap.sh`) — two modes: symlink (local checkout) and
  download (remote `curl | bash`). Installs **only** the minimal global set
  hardcoded at line 271: `GLOBAL_SKILLS=(tailor seed)` plus all agents. The
  per-repo allowlist mechanism (`.spellbook.yaml`) was removed in commit
  `f91f1c4 refactor(harness): pivot to minimal globals` — per-repo work is
  now delegated entirely to `/tailor` and `/seed`, which install into the
  consuming repo's shared root.
- **Scripts** (`scripts/`) — shell + Python utilities: frontmatter check,
  index regen, embeddings, external sync, harness lint, heal/land.
- **Backlog** (`backlog.d/NNN-*.md` open, `backlog.d/_done/` closed).
  `git-bug` is the distributed issue store; `.spellbook/deliver/<ulid>/` is
  `/deliver` runtime state (gitignored).
- **Index** (`index.yaml`) — derived artifact, regenerated by pre-commit.
  Never human-edited.
- **Registry** (`registry.yaml`) — external skill sources for embeddings +
  `scripts/sync-external.sh`; syncs into `skills/.external/` (gitignored).

## Load-bearing gate

**`dagger call check --source=.` IS the gate.** 12 parallel sub-gates:

| Gate | What it enforces |
|---|---|
| `lint-yaml` | All `*.yaml`/`*.yml` parse |
| `lint-shell` | `shellcheck --severity=error` on every non-`ci/` shell script |
| `lint-python` | `py_compile` on every non-`ci/` `.py` |
| `check-frontmatter` | `scripts/check-frontmatter.py` — required fields, line limits |
| `check-index-drift` | `index.yaml` matches `scripts/generate-index.sh` output |
| `check-vendored-copies` | Vendored copies match canonical sources |
| `test-bun` | `bun test` under `skills/research/` |
| `check-exclusions` | No `@ts-ignore`, `.skip()`, `eslint-disable`, `as any` in source |
| `check-portable-paths` | No hardcoded `/Users/<name>/` or `C:\Users\` outside `harnesses/claude/` + `.claude/hooks` |
| `check-harness-install-paths` | `scripts/check-harness-agnostic-installs.sh` — seed/tailor copy must not be Claude-only |
| `check-deliver-composition` | `skills/deliver/SKILL.md` composes atomic phase skills via trigger syntax, never inlines |
| `check-no-claims` | No `claims.sh` / `claim_acquire` / `claim_release` under `skills/` (dropped per backlog.d/032) |

The gate is self-healing: `dagger call heal --source=. --model=gpt-4.1
--attempts=2` spawns a Dagger `llm()` repair agent for one failing
lint-style gate (yaml/shell/python/frontmatter), bounded.

**Pre-commit hook** (`./.githooks/pre-commit`): regenerates `index.yaml` when
`skills/` or `agents/` change; runs `scripts/check-harness-agnostic-installs.sh`;
blocks force-adds of `.spellbook/deliver/<ulid>/(state|receipt).json`.

**Pre-merge-commit hook**: verdict gate for non-FF merges. Escape:
`SPELLBOOK_NO_REVIEW=1`.

**Post-commit / post-merge / post-rewrite**: re-run `./bootstrap.sh` when
skill/agent content changed so global symlinks stay current.

## Invariants

- **Cross-harness first (Red Line).** Every new mechanism — skill, hook,
  setting, lint — works on Claude, Codex, AND Pi. Harness-native runtime
  features are optimizations on top of filesystem + SKILL.md, not the base.
  Prior art: `harnesses/pi/settings.json:skills[]` globs.
- **Skills are self-contained.** No `../..`, no `$REPO_ROOT/…` sourcing.
  Libs resolve via `readlink -f` + `$SCRIPT_DIR/lib/…`. State roots
  (cycles, locks, backlog) anchor to the *invoking* project's
  `git rev-parse --show-toplevel`, not the skill's install dir. Symlink-
  install + invoke from a foreign project is the canonical test.
- **Thin harness, strong models.** Don't compensate for weak models with
  scaffold. Reference form: `skills/flywheel/SKILL.md` (43 lines).
- **Description is the trigger.** Frontmatter `description:` is assertive,
  not aspirational.
- **No `index.yaml` edits.** Pre-commit regenerates it; manual edits are
  churn or drift.
- **No `references/<repo-name>.md` sidecar files.** Repo-specific content
  belongs in SKILL.md body. Stack-specific references under their topic
  (`references/convex-patterns.md`) are fine.
- **`.spellbook/deliver/<ulid>/state.json` + `receipt.json` are agent-
  written, never human-edited, never committed.** Gitignored by default;
  pre-commit hook blocks force-adds.
- **Base branch is `master`.** Topic branches named `feat/*`, `fix/*`,
  `chore/*`, `docs/*`, `refactor/*`.
- **No claim-coordination primitives under `skills/`.** `claims.sh`,
  `claim_acquire`, `claim_release` dropped per backlog.d/032; regression
  guarded by `check-no-claims`.
- **`/deliver` must compose atomic phase skills via trigger syntax** —
  `/code-review`, `/ci`, `/qa`, `/implement`, `/refactor`, `/shape`. Raw
  `dagger call check`, `bunx playwright`, or direct bench-agent dispatch
  inside `skills/deliver/SKILL.md` fails `check-deliver-composition`.
- **`harnesses/claude/settings.json` is COPIED by bootstrap**, not
  symlinked (Claude mutates it at runtime). Changes require re-bootstrap.

## Terminology

- **primitive** — a skill or agent.
- **catalog** — the canonical `skills/` + `agents/` under the spellbook
  repo root. Source of truth.
- **harness** — a specific CLI (Claude Code, Codex, Pi) that hosts skills
  via its own scanning convention.
- **shared root** — `.agents/skills/` in a consuming repo. Canonical install
  location; harness-specific dirs symlink back to it.
- **bridge** — a per-harness symlink layer like `.claude/skills/` pointing
  into the shared root.
- **load-bearing gate** — the one thing that must pass to ship. Here:
  `dagger call check --source=.`.
- **heal** — `dagger call heal` — bounded LLM repair of a single failing
  lint gate.
- **double diamond** — diverge-before-converge twice; applied in `/groom`
  (problem) and `/shape` (solution).
- **loop-core skills** — shape, refactor, code-review, flywheel, groom,
  implement, diagnose. The connective tissue where depth matters most.
- **tailor-owned** — a skill/agent with a `.spellbook` marker whose
  `installed-by: tailor`. `/tailor` re-runs can replace or remove these;
  unmarked content is preserved.
- **inner loop** — `/deliver` composer: ticket → merge-ready via
  /shape → /implement → (/code-review + /ci + /refactor + /qa) clean loop.
- **outer loop** — `/flywheel` cycles: /deliver → /deploy → /monitor →
  /diagnose (on alert) → /reflect.

## Session signal

Recurring user corrections (policy the harness must encode):

1. **"Cross-harness parity is a Red Line."** — Anchoring a design on one
   harness's unique feature is a bug. Fix: emit per-harness artifacts from
   one source, or use the filesystem-level primary layer.
2. **"Thin harness over semantic orchestration."** — Don't compensate for
   weak models with scaffold; 43-line `/flywheel` is the reference.
3. **"Skills encode judgment, not procedures."** — If the model already
   knows how, delete the skill. Regexes over agent prose is a strong smell.
4. **"Evidence-gated shipping."** — No `/qa` evidence = no PR merge.
   Ship step halts until artifacts exist.
5. **"Per-repo tailoring, not general-purpose."** — `/tailor` writes
   repo-specific SKILL.md bodies; generic jacket + appendix is forbidden
   (the sewn-on-sleeve anti-pattern).

Validated patterns the user has ratified:

1. **Dagger as the gate.** `dagger call check` over GitHub Actions —
   local-first, composable, self-healable.
2. **backlog.d/ as the shaped-work store.** Files, not tickets, are the
   unit of shaped work. `_done/` for closed.
3. **Double diamond by default.** Diverge before converge twice —
   `/groom` for problem, `/shape` for solution.
4. **Cross-model second voice.** Thinktank / Gemini / fresh-context
   subagent with a different foundation, not same-model self-debate.
